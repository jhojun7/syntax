html 이란
    웹페이지 콘테츠를 정의하는 언어
    텍스트 에디터에서 작성한 html코드는 웹브라우저를 통해 표시할 수 있다.

#기초문법
    <p></p>
    <br>
    <strong></strong>
    <!-- -->

#문서구조
    <!DOCTYPE html>
        문서 첫 부분에서 문서 유형을 지정하는 단일 태그
    <html> </html>
        문서 유형을 지정한 후 실제 문서가 시작되고 끝나는 것을 나타내는 태그
        내부에 다양한 태그들이 포함되어 문서의 내용을 구성
        <head> </head>
            웹 브라우저 화면에는 보이지 않지만 웹 브라우저가 알아야 할 정보들은 모두 이 태그에 들어간다.
            <meta charset="uth-8">
                문자 인코딩 및 문서 키워드 등에 대한 요약 정보를 기입하는 단일 태그
                영문과 한글을 모두 사용하기 위해 utf-8 방식을 사용하는 것이 좋다.
            <title> <title>
                문서의 제목
                콘텐츠는 브라우저 탭에 표시
        <body> <body>
            실제 브라우저 화면에 표시될 내용을 입력
            텍스트 표시 태그 , 이미지 표시 태그 , 사용자 인터페이스를 나타내는 태그 등 콘텐츠를 다양하게 구성 가능

#텍스트 태그
    <p> </p>
        하나의 p태그는 하나의 문단을 표현
        문단과 문단 사이에는 공백이 있음
#제목 태그
    <h1> </h1> ... <h6> </h6>
        h태그는 제목 요소를 나타내는 태그
        숫자와 함께 사용되며, 숫자1일때 가장 크고 6일때 가장 작다.
        
#수평선 태그
    <hr>
        수평선을 표시하며 단일 태그
        주제 변경 또는 내용 구분등에 주로 사용

#HTML 특징
    HTML코드에서는 줄바꿈인 엔터를 무시, 스페이스 공백도 한번씩만 인정
    따라서 아래의 태그들을 이용
    #줄바꿈 태그
        <br/>
    #공백을 표시
        &nbsp;

#태그의 요소
    #블록레벨 요소
        자기가 속한 영역의 너비를 모두 차지하여 블록을 형성
    #인라인 요소
        자기에게 필요한 만큼의 공간만 차지
        <strong> </strong>
            굵은 글자를 만드는 인라인 태그
        <em> </em>
            기울여진 글자를 만드는 인라인 태그
        <mark> </mark>
            형광펜 효과를 만드는 인라인 태그

#컨테이너 태그
    콘텐츠나 레이아웃에 아무런 영향을 주지 않고, 단지 다른 요소 여럿을 묶어 관리하기 편하게 만드는 역할을 하는 테그
    콘텐츠 내용을 구분하거나, 공통적인 스타일을 적용하고자 할 때 사용하는 것이 좋다.
    #블록레벨 컨테이너
        <div> </div>
    #인라인 컨테이너
        <span> </span>
    
#전역 속성
    모든 HTML 태그에서 공통으로 사용할수 있는 속성
    속성이란, 태그의 부가적인 기능을 정의하는 것으로 선택사항이다.
    속성은 시작태그의 내부에 정의한다.
    속성의 개수에는 특별한 제한이 없다.

    속성을 추가하는 방법
        <태그명 속성명="속성값" 속성명="속성값"> 콘텐츠 </태그명>
    #id
        요소에 고유한 이름을 부여하는 식별자 역할 속성
        태그당 하나씩만 지정
    #class 
        요소를 그룹 별로 묶을 수 있는 식별자 역할 속성
        다중 지정 및 중복 가능
    #style
        요소에 적용할 CSS스타일을 선언하는 속성
    #title
        요소의 추가정보를 제공하는 텍스트 속성. 사용자에게 툴팁 제공
    더 많은 전역속성
        https://developer.mozilla.org/ko/docs/Web/HTML/Global_attributes
    
#링크
    a태그요소는 href 속성을 통해 다른 페이지, 전화번호, 이메일 주소와 그 외 다른 url로 연결 할 수있는 링크를 만든다.
    인라인 요소이며, 콘텐츠는 주로 링크의 목적지를 나타낸다
        /예시 : <a href="https://www.naver.com"> 네이버 </a> (현재 탭에서 열기 : 기본값)
    새 탭에서 열기
        /예시 : <a href="https://www.naver.com" targer="_blank"> 새탭으로 네이버 </a>
        (현재 탭으로 열때에는 target="_self"이지만 기본값.)
    전화번호
        <a href="tel:010-1234-5678"> 010-1234-5678 </a>
    메일
        <a href="mailto:abcde123@naver.com"> abcde123@naver.com </a>

#입력요소 input
    사용자로부터 값을 입력받을 수 있는 대화형 컨트롤을 나타낸다.
    기본적으로 인라인 요소이며, 단일 태그이다.
    type속성의 값에 따라 입력 요소의 형태나 입력 데이터 유형 등이 달라진다.
    사용 가능한 type은 20여 가지이며, 기본값은 text이다.
        사용가능한 모든 속성에 대한 정보
        https://developer.mozilla.org/ko/docs/Web/HTML/Element/Input
            text - 기본값, 텍스트를 입력
            button - 버튼 생성
            color - 팔레트로 색깔 지정
            range - 바? 같은거 생성
            date - 캘린더를 통해 날짜 지정
    name 식별자를 추가할수 있다.
        /예시 <input type="text" name="nickname" />
    텍스트에서 최대입력값을 maxlength 를 통해 지정 가능

#입력요소 select
    다수의 옵션(선택지)울 포함할 수 있는 선택 메뉴.
    메뉴 안에 포함되는 옵션은 option 태그를 사용해 표시
    name 지정할 수 있고 value 속성을 지정할 수 있다. (value는 실제로 처리될 값)
    /예시
        <selcet name="cafe">
            <option value="starbuck"> 스타벅스 </option>
            <option value="coffeebean"> 커피빈 </option>
            <option value="eduya"> 이디야 </option>
            <option value="pascucci"> 파스쿠찌 </option>
        </select>

#입력요소 textarea
    인풋은 한줄밖에 입력 못하는 반면, 여러 줄의 일반 텍스트를 입력 할 수 있는 입력요소
    name 추가하여 구별 할수 있다.
    /예시
        <textarea name="letter"></textarea>

#form태그
    사용자가 입력한 데이터값(입력값)을 서버로 보내기 위해 사용하는 태그
    서버란, 정보를 제공하는 호스트
    /예시
        <form>
            <input type="text" placeholder="ID">
            <br>
            <input type="text" placeholder="PW">
            <br>
            <input type="submit" value="Login">
        </form>
            submit 타입은 입력값을 제출
            로그인 버튼을 누르면 입력된 아이디와 비밀번호가 서버로 전송(요청)
            서버 측에서 데이터를 처리한 결과를 클라이언트에게 보냄(응답)
    
    action : 입력값을 전송할 서버의 url, 서버 측 주소를 지정하는 속성
    method : 클라이언트가 입력한 데이터를 어떤 식으로 전송할지 (GET ot POST), 데이터 전송 방식을 지정하는 속성

    Get : 불러오기. 서버에 요청을 보내어 응답을 받아낸다.
    Post : 추가,수정,삭제. 서버에 요청을 보내어 작업을 수행, 서버에 있는 데이터를 추가, 수정, 삭제 한후 응답을 받아낸다.

#meta태그
    HTML문서에 대한 메타데이터를 정의한다.
    메타데이터란 데이터에 대한 데이터, 즉 정보를 의미한다
    메타태그는 항상 head태그 안에 들가며, 일반적으로 문자세트, 페이지설명, 키워드. 문서의 작성자 및 뷰포트 설정을 지정하는데 사용
    
    메타태그를 사용하는 이유?
        웹페이지에 대한 정보를 제공하므로 검색엔진이 페이지를 검색할 때 참고할 수 있고, 검색결과 반영 할 수 있다.

    #charset : 문자 세트
        문자 인코딩에 대한 요약정보를 기입하는 속성.
        문자 인코딩이란 한글을 표시하기 위해 문자세트를 지정하는 작업으로, 영문과 한글을 모두 사용하기 위해 uth-8방식을 사용하는 것이 좋다.
        명확하게 기입 해두지 않으면, 상황에 따라 자동으로 인코딩을 추정하여 문자가 깨질 가능성이 있다.
    #http-equiv : 콘텐츠 속성 정보에 대한 http 헤더
        콘텐츠 속성의 정보나 값에 대한 http헤더를 제공한다.
        http란 인터넷에서 데이터를 주고 받을 수 있는 프로토콜이다.
    #name : 문서 정보
        속성을 이름으로, 콘텐츠 속성을 값으로 하여 문서 정보를 이름+값 쌍의 형태로 제공할 때 사용할수 있다.
    #content : 메타데이터 내용
        메타데이터의 구체적인 내용을 기입하는 속성

#뷰포트
    현재 화면에 보여지고 있는 영역
    기기별로 다르기때문에 동일한 웹페이지라도 기기에 따른 배율 조정이 발생해 화면의 크기가 다르게 보이는 현상이 나타난다.
    메타태그를 통해 뷰포트 관련 설정을 추가할수 있다.
    /예시
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        * 뷰포트의 너비를 단말기 너비에 맞추고, 초기 배율을 1로 한다
